Toasts must be used carefully, as they may pose problems to accessibility.

We did our best to make it accessible and affordable, but as its message is time-sensitive, it will always violate the WCAG21 2.2.1: Timing Adjustable criteria.

A toast must be a single line of text; any other element inside it will be ignored.

Alert Toast

The "alert" role should be used for alert or error messages only. The main difference with the role "status" is that "alert" will stop the current reading of the screen reader in order to read the toast, while "status" will wait for the end of the current reading before announcing its content.

Properties and Events

Properties applicable to the component c3m-toast
Name Type Default Description
<identifier> * #xxxxx -- Unique identifier of a toast
toastText * string -- Content of the toast
role string "status" Specifies the type of message ("status" or "alert"). "status" should be used to indicate a success message.
delay number 6500 Time in milliseconds to display toast. Toast will not close by itself if value is set to 0.
normal (class) -- Changes style of the toast, to indicate a generic message
styleClass string "slide-to-bottom" Speficies direction of animation ("slide-to-bottom" or "slide-to-top")
disabled string -- Disables close button if value "disabled"

* mandatory properties
