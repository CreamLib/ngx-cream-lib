Adding your own style-sheet

Besides the CreamNg style-sheet, you may add your own and define your css custom properties.

angular.json

  "assets": [
      "src/favicon.ico",
      "src/assets",
      {
        "glob": "**/*",
        "input": "/node_modules/ngx-cream-lib/assets",
        "output": "/assets/"
      }
    ],
    "styles": [
      "/node_modules/ngx-cream-lib/assets/css/main.css",
      "src/assets/css/local.css"
    ],
    "scripts": []
  },

You may group your local css custom properties in a separate file and import it from your css.

Your local CSS and variables will be loaded last, so you may overwrite any style you need to.

Using Icons

This library uses exclusively svg images for their high level of quality, easy manipulation and their light weight.

For maximum flexibility and performance, we use svg images in three different ways:

  • Svg image files: some complex images are stored as svg files in the ./assets/img folder. This folder holds mostly the application logo in different styles;
  • Svg “sprite”: most of the simple icons are listed in what we call a svg “sprite”, which is a list of inline svgs. This list gives us a total flexibility in terms of accessibility and style. The svg sprite sits in the ./assets/img/sprite.svg file, so svg icons can be referred to from anywhere in the application. Notice that, besides the icons of the first level of the main navigation (with Ids “sec-XXXX”) and some exceptions, all other icons are 24 x 24 pixels;
  • Svg backgrounds: some inline svgs are converted to data URI to be used as background images. This technique allows us to change svg images through HTML classes.

The choice of the way we will insert the svg will depend on the role it will have in the application and the kind of style flexibility they will need.

Decorative Images

Pure decorative images — images that don't carry information — may be referred in the svg sprite or encoded in data URI to be used as background. If the icon should be transformed based on the component state, it is easier to do that if the icon is called as background.

For example, in the case of radio or check boxes, the svg images that replace the native elements are stored as background images so they can be easily changed if the radio/check box is selected of not (refer to the radio-item or checkbox components as examples).

If this is not the case — the image will not be transformed by a specific state —, you may use an icon listed on the svg sprite. To do so, you have to reference the svg by its ID, with the use element:

<svg role="img" width="24" height="24" aria-hidden="true">
  <use href="./assets/img/sprite.svg#pic-arrow"></use>
</svg>

Notice that, as the image carries no information, we may hide it from screen readers by setting the aria-hidden attribute as "true".

Non-decorative images

Images that carry information, i.e. they are not purely decorative, must appear as:

  • image files properly referenced by the img element and with a mandatory alt attribute to describe its purpose or meaning;
  • svg sprite image called with the use element, with a mandatory title element describing its meaning or purpose: 
<svg role="img" width="24" height="24">
  <title>Favorites</title>
  <use href="./assets/img/sprite.svg#ic-favorites"></use>
</svg>

Changing styles

Icons listed in the svg sprite don't have a predefined color, so it may be modified by CSS. You can do that by modifiying the styles directly on the svg sprite — changing globally — or you can modify the color on the component's stylesheet; check the main-nav component's stylesheet for an example.

Size can also be modified by CSS using the transform property or width and height properties.

Adding icons to svg sprite or background

Svg images are great because they are purely HTML, what makes its manipulation very easy and flexible.

If you need to add a new icon to the svg sprite, simply get the svg code (without the opening and closing svg tags), group it in a g and give it a unique ID. That's all you have to do to refrence to this icon later in the application.

If you want to add the image as background, you have to encode the svg source into a data URI first.

You may use Yuri Yoksel's URL Encoder for SVG to do that.
