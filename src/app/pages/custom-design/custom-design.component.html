Adding your own style-sheet

Besides the CreamNg style-sheet, you may add your own and define your css custom properties.

In your angular.json file, add the following lines: 


  "styles": [
    "/node_modules/ngx-cream-lib/assets/css/main.css",
    "src/assets/css/local.css"
    
],

You may also group your local CSS custom properties in a separate file and import it from your css.

Your local CSS styles and variables will be loaded last, so you may overwrite any style you need to.

Using Icons

Changing styles

Icons listed in the CreamNg svg sprite don't have a predefined color, so they may be modified by CSS. You can do that by modifiying the styles directly when calling a svg from the svg sprite, or you can modify the color on the component's stylesheet.

Icons that appear in white have the "icon-bt" class, so you may use this class to change its fill.

Size can also be modified by CSS using the transform or width and height properties.

Adding icons to your project

Svg images are great because they are purely HTML, what makes its manipulation very easy and flexible.

Once you've added your own project assets path to angular.json (see Configuring CreamNg for more information), you can create your own svg sprite, as the example here after: 

<!-- SVG DEFINITIONS -->
<svg xmlns="http://www.w3.org/2000/svg">
  <defs>
    <style type="text/css" >
        /* add your own style here */
    </style>

    <g id="nav-blocks">
      <path fill="none" d="M0 0h24v24H0z"/>
      <path d="M3 3h8v8H3V3zm0 10h8v8H3v-8zM13 3h8v8h-8V3zm0 10h8v8h-8v-8z"/>
    </g>
    <g id="ic-download">
      <path fill="none" d="M0 0h24v24H0z"/>
      <path d="M3 19h18v2H3v-2zm10-5.828L19.071 7.1l1.414 1.414L12 17 3.515 8.515 4.929 7.1 11 13.17V2h2v11.172z"/>
    </g>
  </defs>
</svg>

If you need to add a new icon to your svg sprite, simply get the svg code (without the opening and closing svg tags), group it in a g and give it a unique ID. That's all you have to do to reference to this icon later in the application.

If you want to add the image as background, you have to encode the svg source into a data URI first.

You may use Yuri Yoksel's URL Encoder for SVG to do that.
