Introduction

This library uses exclusively svg images for their high level of quality, easy manipulation and their light weight.

For maximum flexibility and performance, we use svg images in three different ways:

  • Svg image files: some complex images are stored as svg files in the library ./assets/img folder. This folder holds mostly the application logo in different styles;
  • Svg “sprite”: most of the simple icons are listed in what we call a svg “sprite”, which is a list of inline svgs. This list gives us a total flexibility in terms of accessibility and style, allowing svg icons to be referred to from anywhere in the application.
  • Svg backgrounds: some inline svgs are converted to data URI to be used as background images. This technique allows us to change svg images through HTML classes.

The choice of the way we insert the svg will depend on the role it will have in the application and the kind of style flexibility they will need.

Refer to the " Custom Design " page to know how to modify CreamNg icons or how to add your own svg sprite.

Decorative Images

Pure decorative images — images that don't carry information — may be referred in the svg sprite or encoded in data URI to be used as background. If the icon should be transformed based on the component state, it is easier to do that if the icon is called as background.

For example, in the case of radio or check boxes, the svg images that replace the native elements are stored as background images so they can be easily changed if the radio/check box is selected or not. If this is not the case — the image will not be transformed by a specific state —, you may use an icon listed on the svg sprite. To do so, you have to reference the svg by its ID, with the use element: 

<svg role="img" width="24" height="24" aria-hidden="true">
  <use href="./assets/img/sprite.svg#pic-arrow"></use>
</svg>

Notice that, as the image carries no information, we may hide it from screen readers by setting the aria-hidden attribute as "true".

Non-decorative images

Images that carry information, i.e. they are not purely decorative, must appear as:

  • image files properly referenced by the img element and with a mandatory alt attribute to describe its purpose or meaning;
  • svg sprite image called with the use element, with a mandatory title element describing its meaning or purpose: 
<svg role="img" width="24" height="24" aria-labelledby="the-title">
  <title id="the-title">Favorites</title>
  <use href="./assets/img/sprite.svg#ic-favorites"></use>
</svg>

The aria-labelledby attribute is needed to ensure that the accessible name of the image will be announced properly by screen readers.

Adding role=img helps identifying the svg as a graphic (and some bug with Safari).
