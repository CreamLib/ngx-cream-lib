Internationalisation

CreamNg components use the i18n system for internationalisation. If your application is not in English, you may follow the steps hereafter to translate the library components:

  1. Get CreamNg internationalisation file on Github: messages.xlf
  2. Place the file in the "src/locale" directory
  3. Copy the file and change the name for you local language, for example "messages. fr .xlf"
  4. Edit this new file with the wanted translation using the <target> element:

For more information on how to merge translations into your application, consult Merge translations into the application of the Angular documentation.
